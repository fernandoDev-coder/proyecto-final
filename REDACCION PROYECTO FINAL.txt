TAKE YOUR SEAT


Justificación y relevancia del proyecto
La idea de crear Take Your Seat surgió al notar que muchas páginas de reservas de cine son poco intuitivas, están limitadas a un solo cine o cadena, y no ofrecen una experiencia verdaderamente unificada. En muchos casos, los usuarios deben ingresar a distintas plataformas dependiendo del cine que desean visitar, lo que no solo es incómodo, sino que también fragmenta la experiencia del usuario.
Take Your Seat nace con la intención de ofrecer una solución moderna, centralizada e interactiva, que permite gestionar múltiples cines desde una misma plataforma. Su arquitectura está pensada para escalar sin restricciones, permitiendo la incorporación de nuevos cines sin necesidad de rehacer el sistema o afectar a los ya existentes. Esto no solo beneficia a los usuarios, que pueden acceder fácilmente a una amplia variedad de funciones y películas, sino también a los administradores de cines, quienes encuentran en la plataforma una herramienta flexible y potente para gestionar sus funciones, salas y reservas.
Al apostar por una experiencia de usuario más fluida y una estructura preparada para el crecimiento, Take Your Seat busca convertirse en una referencia en la gestión digital de reservas de cine.
Objetivos generales
1. Desarrollar una plataforma web interactiva que permita a los usuarios reservar entradas de cine de forma rápida, cómoda y segura.

2. Centralizar la gestión de distintos cines en un único sistema, sin limitaciones en cuanto a la expansión del número de salas o funciones.

3. Optimizar la experiencia del usuario mediante una interfaz clara que facilite la navegación, la búsqueda de películas y la selección de asientos.
Objetivos específicos
   1. Implementar un sistema de autenticación y gestión de roles (usuarios y administradores) con protección de rutas mediante middleware.

   2. Diseñar una base de datos relacional eficiente para manejar usuarios, películas, horarios y reservas sin conflictos ni redundancia.

   3. Crear una interfaz dinámica y responsive utilizando Bootstrap 5 y JavaScript para la selección de asientos y validaciones en tiempo real.

   4. Garantizar la correcta gestión de funciones sin solapamiento de horarios.

   5. Desarrollar funcionalidades administrativas para la creación, edición y eliminación de películas y horarios.

   6. Incluir la generación de informes sobre ingresos, ocupación de salas y número de reservas, con métricas claras para facilitar la toma de decisiones.

Diseño del Sistema
En el diseño del sistema de Take Your Seat, se ha tenido en cuenta la necesidad de ofrecer una plataforma escalable, intuitiva y funcional para la reserva de entradas de cine, tanto para usuarios comunes como para administradores. Se ha optado por una arquitectura cliente-servidor, donde el servidor actúa como intermediario entre la base de datos y la interfaz de usuario, proporcionando servicios web que permiten consultar, modificar y almacenar información relacionada con películas, funciones, reservas y usuarios.
El sistema está diseñado para ser modular y escalable, implementado bajo el patrón de diseño MVC (Modelo-Vista-Controlador). Este enfoque permite separar:
                                                                                                            * Modelo: Lógica de negocio y gestión de datos (películas, horarios, usuarios, reservas).

                                                                                                            * Vista: Interfaz del usuario desarrollada con Bootstrap y JavaScript para facilitar la experiencia visual e interactividad.

                                                                                                            * Controlador: Encargado de gestionar la lógica de las rutas, validaciones, control de acceso y conexión con los modelos.

Esta separación permite un mantenimiento más ordenado del código, facilita la implementación de nuevas funcionalidades (como reportes estadísticos o integración con pasarelas de pago) y hace posible escalar la aplicación para integrar nuevos cines sin necesidad de rehacer el sistema.
Arquitectura de la Aplicación
La arquitectura de Take Your Seat se basa en el patrón MVC (Modelo-Vista-Controlador), lo que permite una separación clara entre la lógica de negocio, la presentación de datos y el control de flujo. Esta estructura modular facilita el mantenimiento, la escalabilidad del sistema y la incorporación de nuevas funcionalidades, como la integración de múltiples cines.
▪ Modelo
                                                                                                               * Encargado de gestionar los datos de la aplicación y aplicar la lógica de negocio.

                                                                                                               * Incluye la gestión de entidades clave como usuarios, películas, horarios, asientos y reservas.

                                                                                                               * Se asegura de evitar solapamientos de funciones y sobre-reservas mediante validaciones en tiempo real.

                                                                                                               * Tecnologías utilizadas: MySQL como motor de base de datos, lógica de negocio implementada en PHP mediante Laravel.

▪ Vista
                                                                                                                  * Responsable de la interfaz con la que interactúa el usuario.

                                                                                                                  * Presenta las funcionalidades disponibles como búsqueda de películas, selección de asientos, y gestión de reservas.

                                                                                                                  * Adaptada para usuarios regulares y administradores, ofreciendo una experiencia fluida en cualquier dispositivo.

                                                                                                                  * Tecnologías utilizadas: HTML5, CSS3, Bootstrap 5 y JavaScript para validaciones dinámicas e interactividad.

▪ Controlador
                                                                                                                     * Actúa como intermediario entre el modelo y la vista.

                                                                                                                     * Procesa las solicitudes HTTP (por ejemplo, hacer una reserva o crear una película), realiza validaciones y gestiona las respuestas.

                                                                                                                     * Controla el acceso a funcionalidades sensibles mediante middleware según el rol del usuario.

                                                                                                                     * Tecnologías utilizadas: Laravel (PHP), con middleware personalizado para la protección de rutas y control de sesiones.

Diseño de la Base de Datos
El sistema utiliza una base de datos relacional, optimizada para manejar múltiples cines, funciones simultáneas y un gran número de reservas sin pérdida de rendimiento. El diseño busca mantener la integridad de los datos y facilitar consultas complejas, como reportes por película o recuento de ocupación.
▪ Entidades principales:
                                                                                                                           * usuarios: información básica y rol (usuario o administrador).

                                                                                                                           * peliculas: título, descripción, género, imagen, etc.

                                                                                                                           * horarios: función específica con fecha y hora.

                                                                                                                           * reservas: asiento, estado, usuario, horario y precio total.

                                                                                                                           * asientos: asociados a una sala por función.

▪ Relaciones destacadas:
                                                                                                                              * Un usuario puede tener muchas reservas.

                                                                                                                              * Una película puede tener varios horarios.

                                                                                                                              * Un horario puede tener múltiples reservas.

                                                                                                                              * Las reservas se asocian a asientos específicos para prevenir duplicaciones.

Modelo Entidad-Relación (ERD)
Se ha diseñado un diagrama E/R que representa gráficamente las entidades del sistema y sus relaciones. Este modelo permite entender con claridad la estructura de datos y cómo interactúan entre sí, siendo esencial para el desarrollo y futuras extensiones del sistema.
                                                                                                                                 * El ERD incluye las tablas: usuarios, peliculas, horarios, reservas, y asientos.

                                                                                                                                 * Refleja relaciones como:

                                                                                                                                    * usuarios → reservas (1:N)

                                                                                                                                    * peliculas → horarios (1:N)

                                                                                                                                    * horarios → reservas (1:N)

                                                                                                                                    * reservas → asientos (1:1 por reserva)

Seguridad
La aplicación incorpora varias medidas de seguridad orientadas a proteger tanto los datos personales de los usuarios como la integridad del sistema:
1. Autenticación segura
                                                                                                                                                                        * El sistema de autenticación se ha desarrollado utilizando las características nativas de Laravel, aprovechando su robusto sistema de autenticación y autorización. Se utilizó el sistema de middleware para proteger las rutas sensibles y la función helper bcrypt() de Laravel para el cifrado seguro de contraseñas. La implementación de roles (usuario y administrador) se realizó de manera efectiva para controlar el acceso a diferentes partes del sistema.

                                                                                                                                                                        * Solo los usuarios registrados pueden realizar reservas y acceder a su perfil personal.

2. Cifrado de contraseñas
                                                                                                                                                                           * Las contraseñas se cifran utilizando bcrypt, lo que asegura que no se almacenen en texto plano.

                                                                                                                                                                           * bcrypt genera un "salt" único por cada contraseña, haciendo que incluso si dos usuarios usan la misma contraseña, los hashes resultantes sean diferentes.

                                                                                                                                                                           * Este enfoque protege eficazmente contra ataques de diccionario o fuerza bruta.

3. Gestión de datos sensibles
                                                                                                                                                                              * Los datos del usuario, como reservas o credenciales, son tratados de forma privada y almacenados de manera segura en la base de datos.

                                                                                                                                                                              * Las rutas sensibles están protegidas por middleware personalizado, que restringe el acceso dependiendo del rol del usuario (usuario o administrador).

Dificultades Encontradas y Cómo se Superaron
Durante el desarrollo de PRfit surgieron varios desafíos técnicos, especialmente al trabajar con funcionalidades completas de backend y la gestión de bases de datos relacionales. A continuación, detallo los problemas principales y cómo fueron solucionados:
                                                                                                                                                                                    * Backend y sistema de autenticación:
Al principio intenté implementar la autenticación con la API de Google, pero requería un sistema de tokens de corta duración que complicaba mucho su configuración y no me daba tiempo para tener algo estable y funcional a tiempo. Por eso decidí implementar yo mismo el sistema de registro e inicio de sesión, usando Node.js, Express.js y bcrypt para cifrar las contraseñas.

                                                                                                                                                                                    * Rutas y guardado de datos:
La estructura de rutas se implementó siguiendo las convenciones de Laravel, lo que facilitó la organización y mantenimiento del código. La gestión de datos en MySQL se realizó de manera eficiente utilizando las migraciones y modelos de Laravel, lo que permitió un control preciso sobre la estructura de la base de datos y las relaciones entre tablas.

                                                                                                                                                                                    * Optimización de CSS:
Al inicio del proyecto, se planteó crear archivos styles.css individuales para cada componente, pero esta aproximación resultaba en código duplicado y dificultaba el mantenimiento. Como solución, se implementó un sistema centralizado utilizando las plantillas de layout y app.css de Laravel, lo que permitió optimizar y reutilizar los estilos en todo el proyecto.
